235-5
Project 5

Finally no more books, this project taught us how to understand and implement doubly linked lists, sorting functions, and comparators. Even though this project was hard I did find it a little fun because of the trial and error aspect that was in it as well as the topic of the project, our task was "The YGOCard is a class representation of a Yu-Gi-Oh! trading card. Don't worry, you won't need to know how to play the game, all you need to know is that a YGOCard has 7 data members: the name of the card, the type of card, the level (or rank) of the card, the attribute of the card (Wind, Earth, Fire, Water), the race of the monster of the card (Machine, Dragon, Fiend) and finally the attack points (atk) and defense points (def) of the monster. Now to keep this project simple we are only concerned with 3 elements: name, atk, and def. When sorting we will use atk or def as the keys so that we are able to sort the cards by attack value. The DoublyLinkedList class that you've seen in lecture and the Node class that goes along with it. These implement a doubly-linked list data structure. The CardShop class inherits from DoublyLinkedList. The parameterized constructor is similar to the one you implemented in the previous project: it will read from an input file and fill up the shop with pointers to the corresponding YGOCard objects (note that it will store pointers to YGOCard objects, not the object itself). The CardShop class extends DoublyLinkedList with 4 sorting algorithms that you will implement in order to sort all the YGOCard objects by metrics such as atk and def.". This was a little hard to understand and took a lot of youtube videos to understand the various sorting techniques but it was fun never the less.
